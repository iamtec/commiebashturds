How To Set Up a Hexxit Server on Ubuntu 14.04

This tool creates a custom script to deploy a brand new Droplet based on the selected DigitalOcean tutorial. You do NOT need an existing Droplet to get started.

When you're done, spin up a Droplet and paste the script into your User Data. That's it! Your new Droplet will be created with the tutorial configuration already done!

Tags: Miscellaneous Distribution: Ubuntu

Introduction

Hexxit is a Minecraft modpack, i.e. a collection of Minecraft mods. The original game has its roots in freeform exploration and building; Hexxit augments this by adding boss fights, new structures to explore, and lots of rare loot.

Prerequisites

To follow this tutorial, you will need:

  • One Ubuntu 14.04 Droplet.

You may want to choose a Droplet with several GBs of RAM if you intend to have a lot of players on your server.

Step 1 - Installing the Necessary Packages

In this step, we will install Hexxit's dependencies.

First, update the package list.

  • apt-get update

Next, install Java, screen, and unzip.

  • apt-get install -y default-jdk screen unzip

Step 2 - Installing Hexxit

In this step, we will download and install Hexxit.

Because Hexxit has a significant number of files, we can avoid cluttering the home directory by creating a new directory for Hexxit.

  • mkdir ~/hexxit

Change into the ~/hexxit directory.

  • cd ~/hexxit

Next, download Hexxit. At the time of writing, the most recent version is v1.0.10, but you can double check the most recent version via the Hexxit homepage. Copy the Server Download link for use in the command below.

  • wget http://mirror.technicpack.net/Technic/servers/hexxit/Hexxit_Server_v1.0.10.zip

Unzip the Hexxit files.

  • unzip Hexxit_Server_v1.0.10.zip

You can now remove the original .zip file.

  • rm Hexxit_Server_v1.0.10.zip

Step 3 - Running Hexxit

In this step, we will launch Hexxit in a screen session, so it will continue running after you disconnect from the server.

You should specify the amount of memory you want Java to use with the -Xmx and -Xms` flags. This depends on the size Droplet you created. Here, we use 2 GB as an example, but you should customize this based on your needs.

You can also specify the name of the screen session; here we call it hexxit.

  • screen -S hexxit -dm bash -c "java -Xmx2048M -Xms2048M -jar Hexxit.jar nogui"

Conclusion

You now have a Hexxit server running and can connect to it using your_server_ip. Happy exploring!
